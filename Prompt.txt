请为我开发一个名为"WeatherOS"的iOS风格天气网页应用，要求如下：

### 1. API 配置模块
- 首次进入时弹出配置面板，要求用户填入和风天气API的 host 和 key（支持存储到 localStorage）
- 配置界面采用iOS模态弹窗风格，带有毛玻璃背景效果

### 2. iOS Web App 设计规范（严格遵循）
- 必须实现全沉浸式体验：状态栏区域沉浸（使用 theme-color 和 apple-mobile-web-app-status-bar-style），底部导航沉浸
- 圆角设计：所有卡片采用 iOS 大圆角（18-22px），禁止出现直角方块
- 使用 SF Pro 字体体系（或系统默认无衬线字体），字号遵循 iOS 动态类型规范
- 整体采用 iOS 天气App的视觉层级：大字号温度显示、渐变背景、分层卡片
- 支持 PWA 模式，可添加到主屏幕全屏运行（standalone 模式）

### 3. 数据展示模块（覆盖和风免费版全部能力）
- **实况天气**：当前温度、天气状况图标、体感温度、湿度、气压、能见度、风向风力
- **24小时逐小时预报**：横向滚动卡片展示，包含温度曲线图（SVG或Canvas绘制）和天气图标
- **3天预报**：每日最高/最低温度、天气图标、风力信息
- **空气质量**：AQI指数、PM2.5/PM10数值、空气质量等级色条
- **生活指数**：舒适指数、运动指数、洗车指数、穿衣指数、紫外线、感冒指数等（和风免费版提供的8项基础指数）
- **天气预警**：如有预警，顶部显示特殊警告横幅（支持多种预警类型和级别颜色区分）

### 4. 可视化要求
- 使用 Chart.js 或 ECharts 绘制温度趋势曲线（24小时和未来3天）
- 所有天气图标使用和风天气官方图标或 Skycons/Weather Icons 动态图标
- 背景根据天气状况和时间自动切换渐变（晴天：蓝白渐变；阴天：灰白渐变；雨天：深蓝灰渐变；夜间：深紫黑渐变）

### 6. 刷新机制（关键性能要求）
- **禁止自动轮询刷新**，必须通过手动触发
- 在页面顶部右下角放置刷新按钮（iOS风格圆形按钮，带旋转动画反馈）
- 下拉手势支持刷新（Pull-to-Refresh），释放时触发API调用
- 显示"最后更新时间"标签，明确告知用户数据时效性
- 本地缓存策略：刷新间隔不少于5分钟，避免误操作频繁调用API

### 7. 本地部署与端口配置（重要）
- 提供一个启动脚本 `start.sh`，自动启动本地 HTTP 服务器并绑定到 **0.0.0.0:8080**（方便后期隧穿）
- 支持以下任一部署方式（按推荐程度排序）：
  1. **Python 方案**（最简单）：`python3 -m http.server 8080 --bind 0.0.0.0`
  2. **Node.js 方案**：使用 `npx serve -l 8080 -s` 或简易 Express 静态服务器
  3. **Caddy 方案**（推荐长期使用）：提供 Caddyfile 配置，自动 HTTPS（本地可配合隧穿）
- 必须在文档中说明：**服务器必须绑定 0.0.0.0 而非 localhost，否则外部隧穿无法访问**
- 提供 `cloudflared tunnel` 快速隧穿命令示例（用户只需替换自己的 tunnel token）

### 8. 技术栈与性能
- 单HTML文件实现（或React/Vue单文件组件），纯前端实现
- 使用 Tailwind CSS 或原生CSS实现iOS设计系统
- 错误处理：API失效或限流时显示友好错误状态（非alert弹窗）
- 响应式适配：从iPhone SE到iPhone 15 Pro Max完美适配，底部安全区域适配（env(safe-area-inset-bottom)）

### 9. 交付要求
- 提供完整可运行的单HTML文件（`index.html`）
- 提供 `start.sh` 启动脚本（含端口检测、自动打开浏览器功能）
- 提供 `TUNNEL.md` 隧穿配置文档，包含：
  - Cloudflare Tunnel 安装命令（`cloudflared`）
  - 本地隧穿启动命令（`cloudflared tunnel --url http://localhost:8080` 或永久隧道配置）
  - 如何绑定自定义域名（`cloudflared tunnel route dns &lt;tunnel-name&gt; &lt;domain&gt;`）
- 包含测试用的示例数据模式（无需真实key即可预览UI效果）
- 代码注释标注所有iOS适配细节和安全区域处理

### 10. 安全与访问控制（可选但建议）
- 在页面头部添加简易访问密码验证（纯前端Base64简单加密即可），防止隧道公网暴露后被他人随意使用API Key
- 或在 localStorage 中存储 key，但页面加载时检查 referer/域名白名单（可选安全层）
